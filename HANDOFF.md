# Echo — Disaster Recovery Chat App

## What is this
Echo is a lightweight iOS disaster recovery app. Its only purpose: if the primary app (Zuhause) and its services go down, Aquila can still talk to Altair through this app.

**Separate basket.** Echo must remain independent from Zuhause. Different repo, different codebase, different failure mode. Do not merge them.

## Project structure
```
~/Developer/Echo/
├── project.yml          ← xcodegen spec (source of truth, DO NOT edit .xcodeproj directly)
├── Echo.xcodeproj       ← generated by xcodegen, regenerate with: xcodegen generate
├── Echo/
│   ├── EchoApp.swift    ← App entry + APNs registration
│   ├── ContentView.swift ← Main view + Color extension
│   ├── ChatView.swift   ← WKWebView + CSS injection coordinator
│   ├── Info.plist
│   └── Echo.entitlements
│   └── Assets.xcassets/  ← App icon (patrol cat, 1024x1024)
```

## How it works
- **Chat**: WKWebView loads `https://altairaquila-hafen.duckdns.org/hafen/`
- **Gateway config**: User taps ⚓ anchor button in the Hafen web UI → navigates to `/gateway/config`
- **Push notifications**: APNs token registered to VPS on launch at `/streifzug/api/push/register`
- **CSS override**: WKNavigationDelegate injects dark neumorphic CSS after page loads (replaces glassmorphism with solid backgrounds + box-shadow)

## Backend architecture (DO NOT MODIFY — read only context)
The Hafen web frontend runs on a VPS. Echo just loads it in a WebView. You do NOT need to modify the backend.

- **Gateway** (port 3476): AI Memory Gateway with tag-based tools (SAVE_MEMORY, SEARCH_MEMORY, EXEC_VPS)
- **Hafen backend** (port 3460): Chat API + config + conversations
- **Caddy**: Reverse proxy at altairaquila-hafen.duckdns.org

## DO NOT TOUCH
- Any VPS files or services
- Any backend code
- Any memory/brain data
- The Zuhause project (separate repo, separate app)
- APNs server-side push logic

## Current known issues
1. **CSS injection may not visually change anything** — the CSS override in ChatView.swift's Coordinator needs testing. The glassmorphism→neumorphism swap should replace backdrop-filter:blur() with solid backgrounds + subtle box-shadows
2. **Scaling on iPhone 15 Pro Max** — safe area handling needs work, top navigation buttons may be unreachable
3. **Token display not visible** — backend returns usage data but frontend (Hafen web) has PWA cache issues preventing display
4. **⚓ gateway config link** — currently uses location.href which works but needs swipe-back gesture support in WebView (allowsBackForwardNavigationGestures = true)

## Design direction
Style: **Dark Soft Neumorphism** (NOT glassmorphism)
- Background: #0e0c0a (dark warm black)
- Card surfaces: #1a1714 (solid, not transparent)
- Gold accent: #c4a882
- Text: #e8dcc8
- No backdrop-filter, no blur, no transparency
- Subtle box-shadows: `3px 3px 8px rgba(0,0,0,0.5), -2px -2px 6px rgba(50,44,36,0.12)`
- Floating tab bar shadow: `0 -3px 12px rgba(0,0,0,0.4)`
- Round corners: 20-24pt for cards, 30pt for navigation

Full UI spec (from GPT-5.5 analysis of reference image) is available — ask Aquila for the detailed document if needed.

## Three features, nothing more
1. **Chat** — WebView → gateway
2. **Push** — APNs → proactive notifications from patrol cat
3. **Tentacles** — EXEC_VPS tag tool (handled by gateway, not by the app)

Keep it light. This is a lifeboat, not a cruise ship.
